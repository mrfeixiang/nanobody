What is a Nanobody?
A nanobody (VHH) is the variable domain of a heavy-chain-only antibody found in camelids (camels, llamas, alpacas). At ~15 kDa, it is the smallest naturally occurring antigen-binding fragment — roughly 1/10th the size of a conventional antibody.

03VHH Sequence Extraction

Motif-driven extraction handles amber stop codons in phage display vectors.

Troubleshooting: ORF Search Failed (0 VHH found!)
Traditional 6-frame ORF search found 0 VHH sequences because an amber stop codon (TAG → *) in the FR2 region truncated every ORF. This is a design feature of phage display vectors — in suppressor strains, TAG is read through as Gln. Solution: Search for conserved VHH motifs (QVQL...WGQG) instead of ORFs, and replace internal stops with X.

08Troubleshooting Log

Real problems encountered during this analysis and how they were solved.

1. SSL Connection Error with NCBI
curl: (56) OpenSSL SSL_read: error:0A000126

Cause: OpenSSL 3.6.2 (conda) incompatible with NCBI's TLS configuration.
Fix: Use ENA API as alternative, or curl -k to bypass SSL verification.
2. VHH Extraction Failed — 0 sequences found
ORF search returned only 6-109 aa fragments instead of expected 120 aa VHH.

Cause: Amber stop codon (TAG → *) in FR2 region truncates ORF.
Fix: Use conserved motif search (QVQL...WGQG) instead of ORF search. Replace internal * with X.
3. IgBLAST Missing D/J Databases
BLAST Database error: No alias or index file found for [human_gl_D]

Cause: conda IgBLAST only includes V gene internal data, not D/J BLAST databases.
Fix: Manually create D/J germline FASTA files and build BLAST databases with makeblastdb.
4. IMGT Database Download Failed
curl returned HTML instead of FASTA from IMGT website.

Cause: IMGT requires browser-based access with authentication.
Fix: Use IgBLAST's built-in internal_data for V genes, manually prepare D/J sequences.
