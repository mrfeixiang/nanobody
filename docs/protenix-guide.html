01Background

Essential knowledge about EGFRvIII, nanobodies, and Protenix v2.

What is EGFRvIII?

EGFRvIII (Epidermal Growth Factor Receptor variant III) is the most common extracellular domain truncation mutant of EGFR. It arises from an in-frame deletion of exons 2-7, removing 268 amino acids (positions 6-273 of wild-type) and inserting a novel glycine at the junction. The resulting receptor is constitutively active, binds no known ligand, and is frequently expressed in glioblastoma (GBM) and other cancers. It is a prime therapeutic target.

What is a Nanobody (VHH)?

Nanobodies are the variable domains of heavy-chain-only antibodies (HCAb) found in camelids. At ~15 kDa, they offer superior tissue penetration, high stability, and can access cryptic epitopes that conventional antibodies cannot reach. They contain 3 CDR loops (CDR1, CDR2, CDR3), with CDR3 being the longest and most critical for antigen binding.

About Nanobody 34E5 (PDB: 8UKV)
The nanobody 34E5 targets Domain IV of EGFRvIII. This epitope is occluded in unliganded intact EGFR but exposed in EGFRvIII due to the truncation, enabling selective binding. This makes it an excellent reference for your designs.

04Approach 1: Structure Prediction with CLI

Use Protenix v2 CLI to predict nanobody-antigen complex structures when you have candidate VHH sequences.

Prepare Input JSON

Create a JSON file specifying the nanobody and antigen chains with optional pocket constraints:

JSON 
[
  {
    "name": "VHH_EGFRvIII_complex",
    "sequences": [
      {
        "proteinChain": {
          "sequence": "YOUR_VHH_SEQUENCE_HERE",
          "count": 1,
          "id": ["H"]
        }
      },
      {
        "proteinChain": {
          "sequence": "EGFRVIII_EXTRACELLULAR_DOMAIN_SEQUENCE",
          "count": 1,
          "id": ["A"]
        }
      }
    ],
    "constraint": {
      "pocket": {
        "binder_chain": { "entity": 1, "copy": 1 },
        "contact_residues": [
          { "entity": 2, "copy": 1, "position": 1 },
          { "entity": 2, "copy": 1, "position": 2 },
          { "entity": 2, "copy": 1, "position": 3 }
        ],
        "max_distance": 6
      }
    }
  }
]

JSON Field Reference

FieldDescription
nameJob name (used for output directory naming)
proteinChain.sequenceAmino acid sequence (standard 20 + X for unknown)
proteinChain.countNumber of chain copies
proteinChain.idChain ID labels
constraint.pocketDefine binding interface between binder and epitope
binder_chainThe binder chain (nanobody)
contact_residuesAntigen epitope residues (1-indexed positions)
max_distanceMaximum contact distance in Angstroms (typically 6)

Generate MSA (Multiple Sequence Alignment)

BASH 
# Generate MSA for sequences
protenix msa --input nanobody.fasta --out_dir ./msa_output --msa_server_mode protenix

# Or use full preprocessing (MSA + template search)
protenix prep --input nanobody_egfrviii_pred.json --out_dir ./prep_output

Run Structure Prediction

BASH 
# Basic prediction with Protenix v2
protenix pred \
  -i nanobody_egfrviii_pred.json \
  -o ./output_prediction \
  -n protenix-v2 \
  -s 101,102,103,104,105

# With template + TFG guidance for enhanced accuracy
protenix pred \
  -i nanobody_egfrviii_pred.json \
  -o ./output_prediction \
  -n protenix-v2 \
  -s 101,102,103,104,105 \
  --use_template true \
  --use_tfg_guidance true

Web Prediction Steps

  1. Open Protenix Server and create a new "Structure Prediction" task
  2. Enter sequences: Chain 1 = VHH nanobody sequence, Chain 2 = EGFRvIII extracellular domain
  3. Select model: protenix-v2
  4. Configure: 5+ seeds, enable MSA search, optionally enable template search
  5. Add pocket constraints to specify target epitope residues (optional but recommended)
  6. Submit job and wait for completion (minutes to hours depending on queue)
  7. Download results: predicted .cif structures + confidence JSON files

Analysis Script

PYTHON 
# analyze_results.py
import json, glob, os

results = []
for conf_file in sorted(glob.glob(
    "output/VHH_EGFRvIII/*/VHH_EGFRvIII_*_summary_confidence_*.json")):
    with open(conf_file) as f:
        data = json.load(f)
    seed = os.path.basename(os.path.dirname(conf_file))
    results.append({
        "seed": seed,
        "iptm": data.get("iptm", "N/A"),
        "ptm": data.get("ptm", "N/A"),
        "ranking_score": data.get("ranking_score", "N/A"),
    })

results.sort(key=lambda x: x.get("ranking_score", 0), reverse=True)

print(f"{'Seed':<8} {'ipTM':<8} {'pTM':<8} {'Ranking':<10}")
print("-" * 36)
for r in results:
    print(f"{r['seed']:<8} {r['iptm']:<8} {r['ptm']:<8} {r['ranking_score']:<10}")

Which Epitope to Target?

Best Choices for EGFRvIII:
  1. Junction region — Unique neoepitope not present on wild-type EGFR, enables high specificity
  2. Domain IV — Exposed on EGFRvIII but occluded on intact EGFR (validated by nanobody 34E5)
Avoid:
Conserved regions shared between EGFRvIII and wild-type EGFR — will cause off-target toxicity.

How to Determine contact_residues?

PYTHON 
# Extract interface residues from reference complex (PDB 8UKV)
from Bio.PDB import PDBParser, NeighborSearch

parser = PDBParser(QUIET=True)
structure = parser.get_structure("complex", "nanobody_EGFRvIII_8UKV.pdb")

nanobody_atoms, antigen_atoms = [], []
for chain in structure[0]:
    for residue in chain:
        for atom in residue:
            if chain.id == "H":
                nanobody_atoms.append(atom)
            else:
                antigen_atoms.append(atom)

ns = NeighborSearch(antigen_atoms)
interface_residues = set()
for atom in nanobody_atoms:
    neighbors = ns.search(atom.coord, 5.0)
    for n in neighbors:
        interface_residues.add(n.get_parent().id[1])

print("Antigen interface residues:", sorted(interface_residues))
